Photo-evidence verification demo

Detects duplicate photos + EXIF/GPS tamper · runs entirely on-device for the hash

Try this: upload any photo, hit Verify (verdict will be ok for the first one). Then upload the same file again → verdict flips to suspect_duplicate (Hamming 0/64). Try a different photo with the same worker ID → that one will be ok. Photos with no EXIF (screenshots, edited images) get a soft flag.

How the verdict is computed

All checks tenant-scoped · last 24 h ledger · in-memory v1

  1. pHash · downscale 32×32 → DCT-II → top-left 8×8 (drop DC) → median threshold → 64-bit hash. Robust to lighting, mild crops, JPEG re-saves.
  2. Hamming dedup · compare to this tenant's last 24 h. ≤5/64 → suspect_duplicate; ≤10/64 → soft flag.
  3. EXIF age · DateTimeOriginal > 24 h ago → suspect_tamper. Missing EXIF → soft flag (likely screenshot).
  4. GPS distance · if photo carries GPS and a claimed location was provided, distance > 5 km → suspect_tamper.

v2 will swap the in-memory ledger for the photo_hashes Supabase table (per-worker indexes, RLS by tenant) and add an Anthropic VLM check that the photo's content matches the task category — pothole, bin, streetlight, etc.
